Using Digital Elevation to create 3D Maps

This was part of a dataset on unusual occurrences during flight for aircrafts approaching to, departing from, or flying near a Canadian airport. The primary data was obtained from the Canadian Open Government Portal and supplemented with the location of Canadian airports. The data was used to locate regions with the highest number of occurrences that lead to serious injuries.

> DEM data was downloaded for Google Earth for the “hottest” locationsn and were then incorporated as layers. Using the HCMGIS plugin, a base map of Google Satellite image was imported. Then the terrain for these regions of interest was mapped into 3D.

The goal was to make an educated guess as to whether the terrain around these airports contributed to the high number of serious occurrences.

NDVI k-means classification

Sentinel-2 data of the KW region from March 1st, 2024, was used to create the k-means clustering. This specific date was chosen there was very low cloud coverage that day, and although there was some snow on the ground, it wasn’t covering the entire landscape. A total of 5 clusters were computed over 7 iterations using the seed 415. The data was then exported to QGIS and overlapped with OSM as the base layer (see corresponding pdf and QGIS project).

It seems clear that the training was clearly able to classify major highways (orange) and snow (purple), and some vegetation (green), but it could not distinguish water (also green) and was not able to separate the local roadways from the houses (both yellow). Perhaps increasing the number of classes or iterations could have improved the classification.

Fastest route between Campuses

This map shows Conestoga College campuses and Grand River Transit routs. GRT Routes used categorized line symbology with the Viridis color ramp to identify each route. The fastest route was computed using Open Route Source plugin, and is a red line at 40% opacity. Various landmarks have been symbolized as points by category, with random colors as the ramp. Finally, the layers for the campuses use solid fill for the symbology.

Using the fastest bus route computed by ORS, the GRT routes that intersect the fastest routes was collected using “Line Intersections” from Analysis Tools. Then the “List Unique Values” item was used to find all the unique GRT routes that intersect the fastest route. The result was the following Route numbers: 10, 16, 36, 57, 61, 67, 76, 110, 201, 203, 206, 302.
